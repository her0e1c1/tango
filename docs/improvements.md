# 改善案一覧

## プロダクト/UX
- 初回利用者向けに、主要機能を短く説明するオンボーディングを追加する。
- 空状態（データがない画面）にサンプル表示と次アクションのガイドを追加する。
- よく使う操作にキーボードショートカットを導入する。

## フロントエンド
- 新規 UI でも `src/features/*` と `src/shared/components` の依存境界を維持し、architecture test を更新する。
- ローディング/エラー状態の表示を共通化してUIの一貫性を高める。
- 画面単位でのコード分割（lazy loading）を進め、初期表示を高速化する。

## 品質/テスト
- 重要ユースケースに対するE2Eテストを追加し、回帰を検出しやすくする。
- flakyになりやすいテストを洗い出し、モックと待機条件を見直す。
- PR時にテストカバレッジの差分を可視化する。

## 開発体験（DX）
- Biomeでformat/lintルールを一元管理し、CIで自動チェックする。
- よく使う開発コマンドを `Makefile` または `npm scripts` に統一する。
- `README.md` に「トラブルシューティング」を追加し、初期セットアップの詰まりを減らす。

## パフォーマンス/運用
- Firestoreアクセスの読み取り回数を計測し、不要クエリを削減する。
- 監視指標（エラー率、描画時間、API応答）を定義して定点観測する。
- 依存パッケージの定期更新フロー（毎月）を決め、脆弱性対応を早める。
